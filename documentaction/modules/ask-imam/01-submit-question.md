# 01. Submit Question

```http
POST /ask-imam
Content-Type: multipart/form-data
Auth: Bearer {{accessToken}} (BROTHER, SISTER)
```

## 1. Overview
Allows a registered user (`BROTHER` or `SISTER`) to submit a new question to the Imam. The user can optionally attach an image to provide visual context. Questions are initially stored with a `pending` status.

> **Response format**: See [Standard Response Envelope](../../README.md#standard-response-envelope)

---

## 2. Business Rules (Source of Truth)

### 2.1 Authentication & Account Status
- **Protected route** — requires a valid `Bearer` token.
- **Role restriction**: Only users with the role `BROTHER` or `SISTER` can submit questions.
- **Status check**: The [auth middleware](../../../src/app/middlewares/auth.ts) automatically blocks access if the user status is `DELETED`, `RESTRICTED`, `SUSPENDED`, `REJECTED`, or `INACTIVE`.
- **Identity**: The `userId` is automatically extracted from the authenticated token and associated with the record.

### 2.2 File Handling (image)
File upload is processed by `fileHandler` **before** validation.

- **Field**: `image` (maxCount: 1).
- **Max file size**: 10 MB (default).
- **Allowed MIME types**: `image/jpeg`, `image/png`, `image/jpg`, `image/webp`.
- **Image processing**: 800px-width resize, PNG palette-compressed level 8, JPEG/WebP quality 80.
- **Multer error mapping** (all -> `400 Bad Request`):
    - `LIMIT_FILE_SIZE` -> `"File too large for field 'image'. Max 10 MB."`
    - `LIMIT_UNEXPECTED_FILE` -> `"Unexpected file field 'X'."`
    - Wrong MIME -> `"Invalid file type 'X'. Allowed for images: [list]"` or `"Unsupported file type 'X'"`

### 2.3 Input Validation (Zod — `submitQuestionZodSchema`)
| Field | Type | Required | Constraint |
| :--- | :--- | :--- | :--- |
| `question` | `string` | Yes | Trimmed; Min 1 char (no whitespace only); Max 2000 chars. |
| `image` | `string` | No | Optional; URL string (attached by `fileHandler` after successful upload). |

### 2.4 Database State & Default Values
- **`status`**: Defaults to `pending` via Mongoose schema.
- **`userId`**: Required; associated with the submitter.
- **`imageUrl`**: Optional; stores the URL generated by `fileHandler`.
- **Atomicity**: No Mongoose transaction is used; `AskImam.create()` is a single operation.

---

## 3. Behavioral Notes

- **Input Sanitization**: `question` is trimmed before validation. Whitespace-only values are rejected by the `.min(1)` rule after trimming.
- **Plain Text Storage**: Question content is treated as plain text; HTML or script tags are not interpreted or rendered.
- **Duplicate Policy**: No submission cooldown or duplicate-question prevention currently exists.
- **Timestamping**: `createdAt` and `updatedAt` are automatically generated by Mongoose timestamps.
- **Orphaned Uploads**: If database insertion fails after a successful upload, the file remains in storage. Cleanup is handled by separate maintenance tasks if applicable.
- **Enum Definitions**: Allowed statuses are `pending` and `answered`.

---

## 4. Request Body (Form-Data)

| Field | Type | Required | Description | Example |
| :--- | :--- | :--- | :--- | :--- |
| `question` | `string` | Yes | The text content of the question (Max 2000) | `How to perform Wudu correctly?` |
| `image` | `file` | No | Optional image attachment (JPG/PNG/WebP) | — |

---

## 5. Implementation
- **Route**: [src/app/modules/ask-imam/ask-imam.route.ts](../../../src/app/modules/ask-imam/ask-imam.route.ts) — `router.post('/', ...)`
- **Controller**: [src/app/modules/ask-imam/ask-imam.controller.ts](../../../src/app/modules/ask-imam/ask-imam.controller.ts) — `submitQuestion`
- **Service**: [src/app/modules/ask-imam/ask-imam.service.ts](../../../src/app/modules/ask-imam/ask-imam.service.ts) — `submitQuestionIntoDB`
- **Validation**: [src/app/modules/ask-imam/ask-imam.validation.ts](../../../src/app/modules/ask-imam/ask-imam.validation.ts) — `AskImamValidation.submitQuestionZodSchema`

**Middleware order**: `auth(BROTHER, SISTER)` -> `fileHandler([{ name: 'image', maxCount: 1 }])` -> `validateRequest(AskImamValidation.submitQuestionZodSchema)` -> `AskImamController.submitQuestion`.

---

## 6. Security
- **Role Enforcement**: Prevents admins or unauthenticated users from submitting questions.
- **File Sanitization**: `fileHandler` ensures filenames are safe and file types are restricted to images.
- **Anti-XSS**: Content is stored as plain text.

---

## 7. Responses

### Success (201)
```json
{
  "success": true,
  "statusCode": 201,
  "message": "Question submitted successfully",
  "data": {
    "_id": "664a1b2c3d4e5f6a7b8c9d0e",
    "userId": "664a1b2c3d4e5f6a7b8c9d0f",
    "question": "How to perform Wudu correctly?",
    "imageUrl": "http://localhost:5000/uploads/images/1715421000-abc123.jpg",
    "status": "pending",
    "createdAt": "2026-05-11T10:30:00.000Z",
    "updatedAt": "2026-05-11T10:30:00.000Z"
  }
}
```

### Error: Validation failed (400)
```json
{
  "success": false,
  "statusCode": 400,
  "message": "Validation Error",
  "errorMessages": [
    { "path": "body.question", "message": "Question cannot be empty" }
  ]
}
```

### Error: Account Suspended (403)
```json
{
  "success": false,
  "statusCode": 403,
  "message": "Account is suspended. Please contact support."
}
```
